功能介绍：一句话想法到可执行需求文档

AI Requirement Studio 面向 AI Coding 场景设计，不是普通“文案生成器”。它的目标是让你把一个模糊想法， 变成一份可以直接给 Cursor、Claude Code、Copilot 等工具继续实现的结构化 Markdown。

要解决的核心问题

  • 想法输入太短，直接让 AI 写代码会反复返工，需求边界经常变化。
  • 需求文档没有验收口径，开发阶段容易“看起来做了，结果不对”。
  • 每次修改需求都要整篇重写，版本很快失控。

核心能力

  • 动态澄清（3-5问）：针对你的输入补齐目标、范围、约束、验收，不再固定模板提问。
  • 质量优先主生成：生成 9 章结构化需求文档，优先可读性和可执行细节。
  • 按标注修订：支持“修改 / 删除 / 补充”三种动作，先命中局部再考虑全篇修复。
  • 全篇修复：你手动触发后，系统对术语、规则、验收口径做一致性联动。
  • 订阅导出：免费可体验生成与修订，导出 `.md` 时按订阅规则校验。

标准使用流程

  1. 输入一句话需求与场景。
  2. 回答关键澄清问题（可跳过非关键项）。
  3. 生成首稿并进入文档编辑。
  4. 在文档中做标注修订，必要时触发全篇修复。
  5. 确认后导出 Markdown 到 AI Coding IDE 执行。

输出文档包含什么

默认输出 9 个章节，重点覆盖：

  • 产品定位、用户画像、核心旅程
  • 功能列表、功能细则、非功能需求
  • 技术架构建议、AI 使用点、商业化路径

每章会尽量包含：实现要点 / 约束规则 / 验收标准 / 默认假设（按信息充分度动态决定）。

适用场景

  • 0-1 产品需求梳理（Web 应用、管理后台、AI 工具、小游戏等）。
  • 需求频繁变更的项目，先稳定文档再生成代码。
  • 外包协作或团队分工，需要统一口径的需求基线。
