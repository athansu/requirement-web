案例 1：SaaS 登录与权限

1) 原始一句话
做一个 SaaS 登录系统。
2) 初版生成片段
功能：支持注册、登录、找回密码。技术建议：使用 JWT。商业化：后续可付费。
3) 修订后片段
功能详细说明（登录） - 输入：邮箱/密码、OAuth 回调 token。 - 系统行为：连续 5 次失败锁定 15 分钟；成功后签发 access+refresh token。 - 约束/规则：Refresh token 30 天有效，重置密码后强制全端下线。 - 验收标准：P95 登录接口 < 400ms；异常提示覆盖 401/403/429。
4) 如何喂给 Codex/Cursor/Claude Code
  • 先粘贴修订后片段，再加一句：按章节编号实现，不得跳过验收标准。
  • 要求先输出 API/DB 变更清单，再提交代码。
  • 把“约束/规则”作为 tests 的断言来源。

案例 2：AI 工具导航站

1) 原始一句话
做一个 AI 工具导航站。
2) 初版生成片段
支持工具列表和详情页，后续接入 SEO。
3) 修订后片段
非功能需求 - SEO：模板页与详情页均为可抓取静态 HTML；每页包含 canonical 与内链。 - 数据质量：重复工具标题 + URL 相似度>0.9 时进入人工审核。 - 验收标准：模板页点击“用此模板生成”到首页预填成功率>99%。
4) 如何喂给 Codex/Cursor/Claude Code
  • 要求按“可抓取静态页”优先实现，不先做 CSR-only。
  • 把“成功率>99%”转成端到端测试目标。
  • 让 AI 输出 sitemap/robots 同步变更。

案例 3：后台审批流系统

1) 原始一句话
做一个带审批的后台系统。
2) 初版生成片段
支持审批和权限管理。
3) 修订后片段
核心用户旅程（审批） - 阶段 A 提交：业务员提交申请，系统校验必填字段并生成待审单据。 - 阶段 B 审批：主管可通过/驳回，驳回需填写原因。 - 阶段 C 归档：通过后写入审计日志（操作者、时间、差异字段）。 验收标准：驳回流程必须保留原数据快照，且支持恢复编辑。
4) 如何喂给 Codex/Cursor/Claude Code
  • 直接让 AI 按阶段 A/B/C 输出状态机与迁移脚本。
  • 要求“驳回原因必填”写入后端校验和前端交互双重约束。
  • 生成审计日志字段的 schema + 索引方案。

案例 4：AI Agent 工作流

1) 原始一句话
做一个 AI Agent 自动化平台。
2) 初版生成片段
支持工作流配置和任务执行。
3) 修订后片段
技术架构建议 - 执行模型：任务进入 queued/running/completed/failed 四态，失败支持最多 2 次指数退避重试。 - 风险控制：涉及外部写操作的节点需人工确认（human-in-the-loop）。 - 可观测性：每个节点记录 requestId、工具入参摘要、耗时与错误码。 验收标准：高风险节点未经确认不得落库。
4) 如何喂给 Codex/Cursor/Claude Code
  • 先让 AI 生成状态机与重试策略，再实现节点执行器。
  • 明确“高风险节点”必须加 gating middleware。
  • 要求输出最小可观测字段并附带日志示例。
